# Практическая работа №5 (CUDA)
## Тема
Реализация параллельных структур данных на GPU с использованием CUDA.

## Цель работы
Целью данной практической работы является изучение принципов реализации параллельных структур данных на графическом процессоре с использованием технологии CUDA. В ходе работы были реализованы параллельный стек и очередь, а также проведено исследование их производительности.

## Что сделано

В данной работе реализованы и исследованы следующие структуры данных:

### 1. Параллельный стек (task1)
- Реализован стек с фиксированной емкостью.
- Операции push и pop выполняются параллельно несколькими потоками.
- Для синхронизации используется атомарный доступ к переменной top.
- Выполнена проверка корректности работы стека.

### 2. Параллельная очередь (task2)
- Реализована очередь с фиксированной емкостью.
- Операции enqueue и dequeue выполняются параллельно.
- Для управления индексами head и tail применяются атомарные операции.
- Выполнено сравнение производительности очереди и стека.

### 3. Дополнительное задание (bonus tasks)
- Реализована очередь с поддержкой нескольких производителей и потребителей (MPMC).
- Для оптимизации используется разделяемая память (shared memory) для подсчёта успешных операций.
- Проведено сравнение производительности GPU-реализаций с последовательными CPU-версиями стека и очереди.

## Структура проекта

- task1.cu — параллельный стек на GPU  
- task2.cu — параллельная очередь и сравнение со стеком  
- bonus_tasks.cu — MPMC очередь, оптимизация памяти и сравнение с CPU  

## Среда выполнения
Код запускался в **Google Colab** с поддержкой CUDA (GPU NVIDIA Tesla T4).

## Результаты
В ходе экспериментов было показано, что:
- атомарные операции позволяют безопасно реализовать параллельные структуры данных;
- использование разделяемой памяти снижает количество обращений к глобальной памяти;
- GPU-реализации демонстрируют преимущество по времени по сравнению с последовательными CPU-версиями при большом количестве потоков.
