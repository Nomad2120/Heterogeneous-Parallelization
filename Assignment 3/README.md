# Assignment 3
## Тема
Архитектура GPU и оптимизация CUDA-программ.

## Цель работы
Целью данного задания является изучение архитектуры GPU и влияния различных способов работы с памятью и конфигураций потоков на производительность CUDA-программ. В рамках работы были реализованы несколько CUDA-программ и проведено сравнение времени их выполнения.

## Что сделано

В работе реализованы **4 отдельных задания**, каждое в своём файле:

### Task 1 — Глобальная и разделяемая память
- Реализовано поэлементное умножение массива на число.
- Сделаны **две версии**:
  - с использованием только глобальной памяти;
  - с использованием разделяемой памяти.
- Проведено сравнение времени выполнения для массива размером **1 000 000 элементов**.

### Task 2 — Сложение массивов и размер блока
- Реализовано поэлементное сложение двух массивов.
- Исследовано влияние размера блока потоков на производительность.
- Проведены замеры для **трёх размеров блока**: 128, 256, 512.

### Task 3 — Коалесцированный и некоалесцированный доступ
- Реализована обработка массива с:
  - коалесцированным доступом к глобальной памяти;
  - некоалесцированным доступом к глобальной памяти.
- Обе версии обрабатывают **одинаковое количество элементов**.
- Выполнено сравнение времени выполнения.


### Task 4 — Оптимизация конфигурации потоков
- Для задачи сложения массивов подобраны:
  - неоптимальная конфигурация сетки и блоков;
  - оптимизированная конфигурация.
- Проведено сравнение производительности и рассчитано ускорение.

## Структура проекта

- `task1.cu` — global vs shared memory  
- `task2.cu` — сложение массивов и block size  
- `task3.cu` — coalesced vs non-coalesced access  
- `task4.cu` — оптимизация grid и block
